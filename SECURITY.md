# 安全策略

## 支持的版本

| 版本 | 支持状态 |
|------|----------|
| 最新 `main` 发布 | ✅ 支持 |
| `dev` 分支 | ⚠️ 开发中，不保证稳定 |

## 报告漏洞

如果你发现安全漏洞，**请勿在公开 Issue 中披露**。

请通过以下方式私下报告：

1. 在 GitHub 仓库使用 **Security → Report a vulnerability** 提交私有报告（**已启用**）。
2. 或通过仓库维护者的私有渠道联系。

### 报告应包含

- 漏洞类型与影响范围
- 复现步骤
- 受影响的分支/版本
- 可能的修复建议（如有）

## 响应时间

- **48 小时内**：确认收到报告
- **7 个工作日内**：初步评估并回复处理计划

## 安全最佳实践（贡献者须知）

- 切勿将 API Key、Token、密码提交到仓库
- 使用 `.env.example` 作为模板，真实密钥放在本地 `.env`（已被 `.gitignore` 忽略）
- CI/CD 密钥使用 GitHub Secrets 管理
- 依赖升级请关注 Dependabot 安全告警
